\section{Objective}
\label{sec:objective}

Now that the main developments shaping reinforcement learning have been outlined, we turn to the formal definition of its objective.  
In this section, we present how reinforcement learning is mathematically formalized.  

To this end, we first define how problems are modeled using Markov Decision Processes (MDPs) in Subsection 1.  
We then introduce the notion of policy, which formalizes the agent’s behavior, and describe how its interaction with the environment is characterized through trajectories and episodes in Subsection 2.  
Finally, we present how policies are evaluated using value functions in Subsection 3.

\subsection{Markov Decision Processes}

In essence, reinforcement learning aims to build a system that makes sequential decisions in order to maximize a cumulative reward signal over the long term.
This learning framework is grounded in the agent paradigm, where the decision-making entity is referred to as the agent.
The agent is characterized by its ability to perceive its environment, process the information it receives, and, in turn, act upon the environment to influence future outcomes.
To formally specify the agent’s task, we model it as a Markov Decision Process (MDP).  

Before providing a more detailed definition of MDPs, it is useful to clarify the scope of the problems considered in this manuscript.  
The class of problems that can be expressed as MDPs is extremely broad, and the agent-based paradigm offers a high degree of flexibility.  
We restrict our analysis to the following configuration:  

\begin{enumerate}
    \item We consider the single-agent learning setting.  
    This means only one agent is actively learning, while the others agents are treated as part of the environment dynamics.  

    \item We focus on finite-horizon MDPs, where interactions between the agent and the environment occur over episodes of fixed length, with a clear beginning and a clear termination.  
    The number of interaction steps may vary, but it is always finite.  

    \item We adopt the partially observable framework, where the agent does not have direct access to the true state of the system, but instead relies on limited observations that provide only partial information about the environment.  
\end{enumerate}

The resulting framework can be formally described as a single-agent finite-horizon partially observable markov decision process.  
However, for the sake of simplicity, we will refer to this setting as an “MDP” throughout this manuscript, unless otherwise specified.  

Now that the scope of the considered problems has been defined, we formally introduce the model.  
A MDP is defined by the tuple \((\mathcal{S}, \mathcal{O}, \mathcal{A}, \mathcal{R}, p)\):  
\begin{itemize}
    \item \textbf{\(\mathcal{S}\)}: The set of states, representing all possible configurations of the environment.
    We denote by \(s^i \in \mathcal{S}\) the initial states, i.e., those in which the agent may begin its interaction, and by \(s^t \in \mathcal{S}\) the terminal states, where the interaction ends.
    
    \item \textbf{\(\mathcal{O}\)}: The set of observations that the agent can perceive from the environment.
    Unlike the true state \(s \in \mathcal{S}\), which is hidden from the agent, the observation \(o \in \mathcal{O}\) provides partial information about the current state.

    \item \textbf{\(\mathcal{A}\)}: The set of actions that the agent may execute.

    \item \textbf{\(\mathcal{R}\)}: The set of rewards that the agent can receive from the environment.

    \item \textbf{\(p\)}: The transition function, which defines the environment's dynamics.
    It is a conditional probability distribution over the next state, reward, and observation, given the current state and action.
    The notation used is \(p(s', r, o \mid s, a)\), and its domain is defined as \(\mathcal{S} \times \mathcal{R} \times \mathcal{O} \times \mathcal{S} \times \mathcal{A} \to [0,1]\).
    This function specifies the joint probability of transitioning to state \(s'\), receiving reward \(r\), and perceiving observation \(o\), after executing action \(a\) in state \(s\).
\end{itemize}

\subsection{Policy}

In a Markov Decision Process, the agent’s decision-making capability is modeled by a policy function.  
The policy defines the mapping between observations and actions, and represents the mechanism through which the agent aims to maximize the cumulative reward over time.  

It is defined as a probability distribution over actions conditioned on observations:
\(\pi(a \mid o)\), for \(a \in \mathcal{A}\) and \(o \in \mathcal{O}\).  
Since it is a probability distribution, \(\pi\) must satisfy the following constraint:
\[
\sum_{a \in \mathcal{A}} \pi(a \mid o) = 1, \quad \forall o \in \mathcal{O}.
\]

Now that both the MDP and the policy have been defined, we can characterize the interaction between them.  
A sequence of consecutive interactions between an environment and a policy is called a trajectory.  

A trajectory is generated through the repeated interaction between the policy and the environment, following the sequence below:
\begin{enumerate}
    \item The environment provides an observation to the policy.
    
    \item Based on this observation, the policy outputs a probability distribution over actions.
    An action is sampled from this distribution.

    \item The action is executed in the environment.
    The environment transitions to a new state and returns a new observation and a reward.
\end{enumerate}

It is important to clearly distinguish between a trajectory as a random variable and its realizations.  
We denote by \(\boldsymbol{\tau}\) the random trajectory.  
For a given initial state \(s_0\), \(\boldsymbol{\tau}\) is a probability distribution over the set of all possible trajectories induced by the policy and the environment dynamics conditioned on \(s_0\).  

A realization of this random variable, obtained by interacting with the environment, is denoted by \(\tau\).  
In other words, \(\tau\) is a sample drawn from this conditional distribution:
\(\tau \sim \boldsymbol{\tau} \mid s_0.\)
For clarity, we use bold notation for the random variable and standard notation for its realizations.  
A realized trajectory of length \(T\) can be written as:
\[
\tau = (s_1, o_1, a_1, r_1, s_2, o_2, a_2, r_2, \ldots, s_T).
\]

There exist two special types of trajectories.
First, a trajectory that terminates in a terminal state is called a terminal trajectory.  
It is denoted by \(\tau_t \sim \boldsymbol{\tau}_t \mid s_0\), where \(\boldsymbol{\tau}_t\) denotes the corresponding distribution over terminal trajectories conditioned on the initial state \(s_0\).
Second, a trajectory that starts from an initial state and is considered as a complete interaction sequence is called an episode.  
It is denoted by \(\tau_e \sim \boldsymbol{\tau}_e\), where \(\boldsymbol{\tau}_e\) represents the distribution over episodic trajectories.

\subsection{Falues fonctions}

Our objective is to learn a policy \(\pi\) that maximizes the cumulative reward within the MDP.  

To evaluate this objective, we introduce the notion of return.  
The return of a trajectory, denoted by \(g(\tau)\), is defined as the sum of rewards collected along the trajectory:
\[
g(\tau) = r_1 + r_2 + \cdots + r_T = \sum_{i=1}^{T} r_i
\]

In reinforcement learning, it is possible to define functions that estimate the expected return given the current state of the system.  
These functions are called value functions.  

The first one we introduce is the state-value function \(V\), defined as follows:
\[
V(s_0) = \mathbb{E}\left[g(\tau_t)\right], \quad \text{with } \tau_t \sim \boldsymbol{\tau}_t \mid s_0.
\]
